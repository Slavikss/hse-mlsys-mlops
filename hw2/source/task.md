Как и в предыдущем домашнем задании, в задании не требуется использовать тяжелые архитектуры моделей. Достаточно упаковать облегченную версию. Цель задания – получить практику с Docker и продуктивизацией решений.

Требования к реализации:
•	Сервис скоринга должен выполнять следующие этапы ML проекта внутри контейнера отдельными скриптами:
o	Чтение сообщений из топика Kafka и выгрузка скора модели и флага фрода в топик Kafka
o	Препроцессинг данных в соответствии с логикой, реализованной вами в соревновании
o	Скоринг обработанного сообщения

•	Inference должен быть реализован только на CPU, GPU будет недоступен.
•	Полученный сервис необходимо оформить в виде Github репозитория с оформленным Readme файлом и подготовленными requirements и docker-compose.yml для сборки docker images и поднятия контейнеров
